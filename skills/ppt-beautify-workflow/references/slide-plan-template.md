# Slide Plan Template

Use this template to map content to one primary slide type before building or rebuilding a deck.

```markdown
# Slide Plan

## Deck Summary
- Total slides:
- Core storyline:
- Must-keep slides:
- Slides likely to split:

## Slide Map
| Slide | Objective | Primary Type | Key Content Blocks | Notes |
|------:|-----------|--------------|--------------------|-------|
| 1 | Cover the deck | cover | title, subtitle, date, presenter | keep sparse |
| 2 | Set the agenda | agenda | 3-5 sections | no dense prose |
| 3 | Introduce section 1 | section | section title, one visual motif | strong contrast |
| 4 | Explain the core argument | single-column | headline, 3 bullets, takeaway | prioritize hierarchy |
| 5 | Compare two options | comparison | option A, option B, recommendation | avoid third column |
| 6 | Show metrics | cards | 3-4 KPI cards, one short caption | large numerals |

## Page Type Rules
- `cover`: sparse, high contrast, one focal idea
- `section`: reset the visual rhythm, do not crowd
- `single-column`: best for one narrative thread
- `two-column`: use only for exactly two grouped ideas
- `comparison`: use for alternatives, before/after, pros/cons
- `timeline`: use for sequential stages or dated events
- `table`: only when truly tabular
- `cards`: use for parallel items or KPIs
```
