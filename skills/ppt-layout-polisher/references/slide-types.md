# Slide Types

Use one primary slide type per page.
Do not mix multiple slide archetypes on the same page unless the user explicitly asks for a custom composition.

## Cover

Use for opening slides with a strong title and limited supporting text.
Reserve generous whitespace.
Center or left-align the hero block and keep supporting metadata compact.

## Section

Use to separate chapters or transitions.
Use a short title and optional one-line descriptor.
Allow more whitespace than a content slide.

## Agenda

Use for lists of upcoming sections.
Prefer one aligned list or two balanced columns.
Avoid paragraphs.

## Single-Column Content

Use for one major argument with a clean reading flow.
Place title at top, then one main content region, then optional source line or note.

## Two-Column

Use for balanced side-by-side material.
Keep both columns visually comparable.
If one side is much taller, change the slide type instead of forcing symmetry.

## Image-Text

Use when one visual and one explanatory block must coexist.
Give the visual enough area to matter.
Do not compress the image to protect too much text.

## Comparison

Use for `A vs B`, `before vs after`, or option comparisons.
Mirror the structure across both sides.
Keep headings, metrics, and bullets aligned row-by-row.

## Timeline

Use for sequences, steps, or milestones.
Limit the number of nodes so labels remain legible.
If milestones become dense, split into multiple slides.

## Table

Use when precise row-column comparison is necessary.
Prefer fewer rows with stronger highlighting.
If the table needs very small text, redesign it into cards or split it.

## Cards

Use for 3 to 6 parallel ideas.
Keep card dimensions, padding, and heading positions consistent.
Do not let one card become a paragraph dump.

## Summary

Use for end pages, conclusions, or key takeaways.
Prefer 3 to 5 concise points with clear hierarchy.
